Longest Continuous Increasing Subsequence

Problem Statement

Given an unsorted array of integers nums, return the length of the longest continuous increasing subsequence (i.e. subarray). The subsequence must be strictly increasing.

A continuous increasing subsequence is defined by two indices l and r (l < r) such that it is [nums[l], nums[l + 1], ..., nums[r - 1], nums[r]] and for each l <= i < r, nums[i] < nums[i + 1].

Example 1:

Input: nums = [1,3,5,4,7]
Output: 3
Explanation: The longest continuous increasing subsequence is [1,3,5] with length 3. Even though [1,3,5,7] is an increasing subsequence, it is not continuous as elements 5 and 7 are separated by element 4.

Example 2:

Input: nums = [2,2,2,2,2]
Output: 1
Explanation: The longest continuous increasing subsequence is [2] with length 1. Note that it must be strictly increasing.

Constraints:

  • 1 <= nums.length <= 104
  • -109 <= nums[i] <= 109

Understanding the Problem

This problem asks for the length of the longest continuous increasing subsequence. Unlike the standard Longest Increasing Subsequence problem, this one requires the elements to be consecutive in the original array.

Key points to understand:

  • The subsequence must be contiguous (consecutive elements in the original array)
  • The subsequence must be strictly increasing (each element must be greater than the previous one)
  • We need to return the length of the longest such subsequence

For example, in [1,3,5,4,7], there are several continuous increasing subsequences:

  • [1,3,5] - length 3
  • [4,7] - length 2

The longest has length 3, so the answer is 3.

Approach

Since we're looking for a continuous subarray where elements are strictly increasing, we can use a linear scan approach.

  • We iterate through the array once from left to right
  • For each element, we check if it continues the increasing trend
  • If it does, we extend our current subsequence
  • If not, we start a new subsequence from the current element
  • Throughout the process, we keep track of the maximum length found so far

Visualization

Let's visualize the algorithm with example [1,3,5,4,7]:

Initialize maxLen = 0, startPos = 0
- Process 1: startPos = 0, current length = 1, maxLen = 1
- Process 3: 3 > 1, continue, current length = 2, maxLen = 2
- Process 5: 5 > 3, continue, current length = 3, maxLen = 3
- Process 4: 4 < 5, break sequence, startPos = 3, current length = 1, maxLen = 3
- Process 7: 7 > 4, continue, current length = 2, maxLen = 3

Final maxLen = 3

Implementation

def findLengthOfLCIS(nums):
    maxLen, startPos = 0, 0
    for i in range(len(nums)):
        if i and nums[i-1] >= nums[i]:
            startPos = i
        maxLen = max(maxLen, i - startPos + 1)
    return maxLen

Time Complexity Analysis

Time Complexity: O(n)

  • We iterate through the array once, which is O(n)
  • Each operation inside the loop takes constant time
  • Therefore, the overall time complexity is O(n)

Space Complexity: O(1)

  • We only use a constant amount of extra space (variables maxLen and startPos)
  • Therefore, the space complexity is O(1)
